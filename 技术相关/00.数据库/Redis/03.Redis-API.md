### 常用全局命令
命令|说明|时间复杂度|备注
-|-|-|--
keys [pattern]|查看所有键|O(n)|如果key很多，可能执行时间较长，慎用; pattern 正则
scan cursor [match pattern] [count number]|渐进式遍历键||match pattern 模式匹配；count number 默认为10，每次遍历的键个数
dbsize|获得键总数|O(1)|
exists key|检查键是否存在||
del kye [key ...]|删除键
expire key esconds\|timestamp|设置键过期时间||persisit命令可以清除键的过期时间；对于字符串类型，set命令会默认清除键过期时间
ttl key|查看键过期时间||返回值（大于0：键过期时间；-1：键没有设置过期时间；-2：键不存在
type key|键对应的值得数据结构类型
object encoding key|键对应的值得内部结构编码
rename key newkey|键重命名
randomkey|随机返回一个键

appent key values|在字符串尾部追加值|O(1)
strlen key|获取长度|O(1)
setrange key offset value|设置指定位置字符|O(1)
getrange key start end|获取部分字符串|O(n)

命令|说明|时间复杂度|备注

hgetall key|获取所有的field-value|O(n)
hmget field [field ...]|批量获取值|O(k)
hmset key field [key field ...]|批量设置值|O(k)
hexists key field|判断field是否存在|O(1)
hkeys key|获取所有filed|O(n)
hvals key|获取所有value|O(n)
hincrby key field increment|自增|O(1)
hincrbyfloat key field increment|自增float|O(1)
hstrlen key field|计算value字符串长度|O(1)
hsetnx key field value|设置值，存在报错|O(1)

### 列表
一个列表最多存储2^32-1个元素，利用提供的方法，可以充当列表或栈来使用

命令|说明|时间复杂度|备注
-|-|-|--
rpush key valule [value ...]|从右边插入元素|O(k)
lpush key valule [value ...]|从左边插入元素|O(k)
linsert key before\|after pivot value|想某个元素前或者后插入元素|O(n)
lrange key start end|查找指定范围内元素列表，包含end本身|O(s+n)
lindex key index|获取列表指定索引下的元素|O(n)
llen key|获取列表长度|O(1)
lpop key|从列表左侧弹出元素|O(1)
rpop key|从列表右侧弹出元素|O(1)
lrem count value|从列表中找到等于value的元素删除，count>0，从左到右，删除最多count个元素；<0，从右到左；=0，删除所有|O(n)
ltrim key start end|按照指定范围修剪列表（只保留指定的）|O(n)
lset key index value|修改指定索引下标的元素|O(n)
blpop key [kye ...] timeout| 阻塞操作，从左侧弹出|O(1)|如果是多个key，会从左至右遍历键，一旦有一个键可以弹出元素，客户端立即返回；如果多个客户端对同一个键进行blpop，那么最先执行命令的客户端可以得到返回的值
brpop key [kye ...] timeout| 阻塞操作，从右侧弹出|O(1)

### 集合：没有重复成员
命令|说明|时间复杂度|备注
-|-|-|--
sadd key element [element ...]|添加元素，返回结果为添加成功的元素个数|O(k)
srem key element [element ...]|删除元素，返回结果为删除成功的元素个数|O(k)
scard key|计算元素个数|O(1)
sismember key element|判断元素是否在集合中|O(1)
srandmember key [count]|随机从集合中返回指定个数元素|O(count)
spop key|从集合中随机弹出元素|O(1)
smembers key|获取所有元素|O(n)
sinter key [key ...]|求交集|O(m*k)
sunion key [key ...]|求并集|O(k)
sdiff key [key ...]|求差集|O(k)

将交集、并集、差集计算的结果保存
```
sinterstore destination key [key ...]
sunionstore destination key [key ...]
sdiffstore destination key [key ...]
```

### 有序集合
保留了集合不能有重复成员的特性，但不同的是，他给每个元素一个分数（score）作为排序的依据。

命令|说明|时间复杂度|备注
-|-|-|--
zadd key score member [score member ...]|添加成员|O(K*log(n))
zcard key|获取成员个数|O(1)
zscore key member|获取某个成员的分数|O(1)
zrank\|zrevrank key member|计算成员排名（从低到高，从高到低）|O(log(n))
zrem key member [member ...]|删除成员|O(K*log(n))
zincrby key increment member|添加成员的分数|O(log(n))
zrange\|zrevrange key start end [withscores]|返回指定排名范围的成员|O(log(n)+k)
zrangebyscore key min max [withscores];zrevrangebyscore key max min [withscores]|返回指定分数返回的成员|O(log(n)+k)|
zcount key min max|返回指定分数范围成员个数|O(log(n))
zremrangebyrank key start end|删除指定排名内的元素|O(log(n)+K)
zremrangebyscore key min max|删除指定分数范围内的元素|O(log(n)+K)
zinterstore destination numkeys key [key ...]|交集|O(n*k)+O(m*log(m))
zunionstore destination numkeys key [key ...]|并集|O(n)+O(m*log(m))
