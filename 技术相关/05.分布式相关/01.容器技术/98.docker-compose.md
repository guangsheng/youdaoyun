一个自动化工具，通过读取配置文件中的信息来组装docker run命令，实现docker容器的自动化启动或停止，减少每次输入的麻烦。

### 配置文件样例，可以配置多个容器
```docker-compose.yaml
root@bogon:/opt/docker# cat docker-compose.yaml 
mytestident:
    build:  /opt/docker/identidock/
    ports:
     - "5001:5000"
mytestredis:
    image: "redis:latest"
```

### 命令说明
- docker-compose COMMAND
    - up 启动在compose文件中定义的容器
    - stop 停止容器，但不会删除它们
    - build 重新创建由Dockerfile构建的镜像
    - rm 删除已停止的容器
    - run 启动一个容器
    - ps 获取由Compose管理的容器的状态信息
    - logs 汇集日志，彩色输出
