#### docker run
```
docker run -t -i ubuntu /bin/bash

```

- docker run是运行容器的指令
- ubuntu 是要运行的image的名称，也可以指定image id
- -t 标识在新容器内分配虚拟终端或终端
- -i 通过抓取容器的（STDIN）来进行交互式连接
- -a 指定数据流（stdout和stderr）连接到容器，与-d选项不兼容
- -d 容器在后台运行，命令返回值时容器ID
- --restart 配置在什么情况下重启容器(no, on-failure[:max-retry], always, unless-stopped)
- --rm 退出时自动删除容器，与-d选项不兼容
- -e, --env 设置容器内环境变量
- -h, --hostname NAME 设置容器的UNIX主机名
- -v, --volume 设置数据卷
- --volumes-from 设置数据卷
- --expose 指定容器将会使用的端口或端口范围
- --link 建立一个与指定容器连接的内部网络接口
- -p, --publish 发布端口（可以通过docker port命令查看分配的端口）
- --entrypoint 覆盖Dockerfile中的ENTRYPOINT命令
- -u, --user   覆盖Dockerfile中的USER指令
- -w, --workdir 覆盖Dockerfile中的WORKDIR指令

##### 运行一个可交互的容器
```
docker run -t -i ubuntu /bin/bash
```

##### 发布端口
```docker run -d -p 8000:80 nginx```  
将主机的8000端口转发到容器80端口
